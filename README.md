## Quick Start

```bash
npx -y symphifony --port 4040
```

Open your browser:

- **`http://localhost:4040`** — dashboard with live queue, events, and sessions
- **`http://localhost:4040/docs`** — full interactive API docs (auto-generated by `s3db.js` ApiPlugin)

That's it. The current directory is your workspace. State lives in `.symphifony/`.

---

## What Happens

1. You create an issue (dashboard, `curl`, or MCP tool)
2. Symphifony routes it to the right agents based on labels, paths, and workflow rules
3. Agents plan, execute, review, retry if needed, and complete
4. You see everything — events, sessions, diffs, provider usage — in real time

```bash
curl -X POST http://localhost:4040/issues/create \
  -H 'content-type: application/json' \
  -d '{
    "title": "Harden websocket reconnect flow",
    "labels": ["backend", "protocol"],
    "paths": ["src/protocol/session.ts", "src/api/ws-handler.ts"]
  }'
```

## Mixed-Agent Pipelines

The killer feature: chain different AI providers in a single pipeline.

```yaml
# WORKFLOW.md frontmatter
agent:
  max_turns: 4
  providers:
    - provider: claude
      role: planner
    - provider: codex
      role: executor
    - provider: claude
      role: reviewer
```

Claude plans. Codex executes. Claude reviews. Each agent gets a hydrated profile,
structured handoff, and full session history.

## What's Inside

| Area | What it does |
|:-----|:-------------|
| **Runtime** | Schedules issues, tracks retries, persists state via `s3db.js`, handles graceful shutdown |
| **Routing** | Uses labels, paths, overlays, and priority rules to pick providers and roles automatically |
| **Agents** | Runs `codex` and `claude` with profile hydration, structured results, and multi-turn sessions |
| **Dashboard** | Live queue, issue details, event timeline, session inspector, runtime health, provider stats |
| **API** | Full REST surface with OpenAPI docs — issue lifecycle, state snapshots, events, sessions, pipelines |
| **MCP** | Expose Symphifony as tools for Claude Code, Cursor, Windsurf, or any MCP-compatible editor |

## Dashboard & API

Start with `--port` and you get both the browser UI and the HTTP API:

```bash
npx -y symphifony --port 4040
```

The API docs at `/docs` are auto-generated from the `s3db.js` `ApiPlugin` resource
definitions — they're always in sync with the actual routes and schemas. No
hand-maintained OpenAPI spec to go stale.

## MCP Server

Run Symphifony as an MCP server over stdio for editor and assistant integrations:

```bash
npx -y symphifony mcp
```

Add to your `.mcp.json`:

```json
{
  "mcpServers": {
    "symphifony": {
      "command": "npx",
      "args": ["-y", "symphifony", "mcp", "--workspace", "/path/to/repo"]
    }
  }
}
```

Now Claude Code (or any MCP client) can create issues, check status, list issues
by state or capability, resolve routing, and review workflows — all without
leaving the editor.

## WORKFLOW.md

Drop a `WORKFLOW.md` in your repo root and Symphifony reads its YAML frontmatter
to configure everything: agent providers, routing priorities, hooks, polling,
concurrency, timeouts, and more.

```yaml
---
agent:
  providers:
    - provider: claude
      role: planner
    - provider: codex
      role: executor
    - provider: claude
      role: reviewer
  max_concurrent_agents: 2
  max_attempts: 3

routing:
  priorities:
    security: 0
    bugfix: 1
    backend: 2
  overrides:
    - match:
        labels: ["frontend"]
        paths: ["src/dashboard"]
      providers:
        - provider: claude
          role: planner
          profile: agency-ui-designer
        - provider: codex
          role: executor
          profile: agency-frontend-developer

server:
  port: 4040
---

You are an expert engineer working on this codebase...
```

The Markdown body becomes the system prompt, available to agents via
`SYMPHIFONY_PROMPT` and `SYMPHIFONY_PROMPT_FILE`.

If no `WORKFLOW.md` exists, Symphifony auto-detects available providers and uses
sensible defaults.

## Run Modes

```bash
# Local runtime (headless scheduler)
npx -y symphifony

# Dashboard + API on a port
npx -y symphifony --port 4040

# MCP server over stdio
npx -y symphifony mcp

# Custom persistence root
npx -y symphifony --persistence /path/to/state
```
